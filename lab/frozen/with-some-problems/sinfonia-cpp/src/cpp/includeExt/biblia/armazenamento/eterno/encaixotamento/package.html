Encaixotamento

Para informações compostas é adequado que haja um "encaixotamento" para distribuir uma mesma informação em diversos pacotes. Um Encaixotador se utiliza de um Empacotador para empacotar/desempacotar cada elemento separadamente da informação composta.

Seu uso é opcional. Porém, algumas informações complexas, geralmente as compostas de mais de uma informação, ou informações que podem se tornar um pacote muito grande, podem funcionar melhor se distribuídas em diversos pacotes. É preciso ponderar se o tempo gasto encaixotando (que é um passo além do de empacotar) é menor que o tempo econimizado na distribuição dos pacotes.

Geralmente o ganho final na distribuição dos pacotes é significativamente maior que se uma informação composta fosse transformada num pacote inteiriço. Pense numa lista, por exemplo: se transformada num único pacote, cada pacote que fosse modificado, o pacote como um todo teria que ser rearranjado. Porém, se cada elemento for distribuído num pacote separado, se apenas um elemento da lista é modificado, apenas esse pacote será modificado.
